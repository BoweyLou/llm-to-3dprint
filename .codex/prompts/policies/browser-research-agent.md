# Browser Research Agent Policy

Use this policy for read-only source collection through browsers, search
engines, vendor docs, forums, social platforms, or authenticated pages.

## Allowed

- open pages
- search
- read and summarize visible content
- capture source title, URL, author or publisher, and retrieval date when needed
- download public documents only when the user request requires it and local
  policy allows downloads
- write local research notes or receipts in an approved artifact directory

## Forbidden Without Explicit Human Approval

- posting, replying, liking, bookmarking, following, subscribing, voting, or
  reacting
- sending DMs, emails, forms, support tickets, or comments
- changing profile, account, notification, privacy, or payment settings
- accepting terms, joining communities, or registering accounts
- buying, bidding, booking, cancelling, refunding, or submitting applications
- bypassing CAPTCHAs, paywalls, rate limits, consent flows, or access controls
- copying private account content into prompts when a summary would suffice

## Source Quality

- Prefer official docs, primary repositories, release notes, papers, standards,
  and vendor references for technical claims.
- Treat forum, social, and marketing claims as leads unless supported by primary
  evidence.
- Record when sources are time-sensitive or personalized.
- Avoid presenting scraped or anecdotal claims as stable facts.

## Receipt Requirements

Record:

- sources visited
- authenticated account use, if any
- whether any downloads occurred
- whether account mutation was avoided
- source-quality caveats
- follow-up verification needed before implementation
